§ III

Per-step detail

What each phase is for
Q
QuestionsPhase one
Resolve ambiguity before R wastes effort.
  • Tiny input, errors compound downstream
  • Frontier-low usually enough; bump if reqs are messy
  • Don't outsource judgment — review the questions
AvoidLetting Q smuggle in design decisions.
R
ResearchPhase two
Evidence collection. No opinions yet.
  • Volume work — bottleneck is recall, not depth
  • Low effort + good tool use beats high effort
  • Tight R compresses the design space for D
AvoidUsing 1M context to skip targeted search.
D
DesignPhase three
Decide solution shape with you in the loop.
  • Dialogue, not delegation — you steer
  • Demand options before recommendations
  • Two-pass (proposer + critic) on risky calls
AvoidPremature convergence on one option.
S
StructurePhase four
File layout and phase ordering given D's call.
  • Highest-leverage planning step — never cheap out
  • Small output, big consequence on rework cost
  • If S struggles, D didn't decide enough
AvoidDropping below high effort, ever.
P
PlanPhase five
Sequenced steps with validation criteria.
  • Narrower search space than S — operates inside it
  • Must include rollback / failure criteria for risky work
  • Good P makes I cheap; bad P makes I expensive
AvoidPlans without explicit validation steps.
I
ImplementPhase six
Execute the agreed plan; report violations.
  • If D/S/P are good, this is mechanical
  • Contract: follow plan; if wrong, stop and report
  • Escalate only when tests turn into diagnosis
AvoidLetting I reinterpret or improvise on the plan.

The frame

Use 1M when many live relationships must be held simultaneously. Not when you want the model to think harder.

Failure modes

  • Stale assumptions anchor
  • Signal drowns in noise
  • Summarizing over deciding
  • Bad context preserved

§ VII

Core principles

Six axioms
01
Capability ≠ effort. Frontier-low often beats weaker-medium at the same cost.
02
Start fast, escalate on signal. Specific trigger, specific step. No blanket bumping.
03
Speed enables iteration. Fast high-quality loops beat one slow perfect pass.
04
D, S, P are the floor. Never cheap out on the planning steps.
05
1M = working memory. Not more thinking. Hold relationships, don't replace reasoning.
06
Wanting 1M downstream = lossy upstream. Fix the handoff, not the context window.
§ VIII

A short list of don'ts

Common failures, named

Workflow

  • Skip D, then Q or S takes its job
  • Same dial across all six steps
  • Bump everything when one step struggles
  • Skip handoff artifacts

Model picks

  • Use max as routine planning
  • Use frontier-low on Opus (respects strictly)
  • Prefer weak-medium over frontier-low on Codex
  • Treat D as delegation, not dialogue

Context

  • Load 1M by default
  • Use 1M to mask weak R
  • Trust opusplan for big-context planning
  • Let I reinterpret the plan
